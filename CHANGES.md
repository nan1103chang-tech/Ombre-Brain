## 一句话介绍

这是 Ombre-Brain 的一个**分支版本**。核心记忆机制——衰减公式、做梦、feel、记忆桶、情感权重——和原作者一致、未改动。本 fork 的改动集中在**前端体验和工具链**，外加一组**检索/排序的命中精度**调优（尤其中文场景）。改了哪些、为什么，见下面第二部分的透明披露。

---

## 与上游的关系

- 核心机制（衰减、做梦、feel、记忆桶、情感权重）与原作者一致；改动集中在前端与检索精度。
- prompt 可在配置页一键对齐原作者版本；字段层兼容原版数据，也可回退上游。
- 会跟进上游更新、取其中有用的部分合入（可能有延迟）。
- 发现的 bug 在力所能及范围内修复。

# 第一部分：网页界面

## 1. 六个独立视图

按用途拆成 6 个独立视图：

### 🔲 记忆格 `/v2/cells/`
记忆的主视图，主打陈列、筛选、查看。所有记忆铺成卡片或列表。
- 多种排序（重要度 / 时间 / "活跃度"score）
- 多种分组（按状态 / 按主题 / 不分组）
- 筛选（钉决 / Feel / 已内化 / 噪声 / 主题域 / 标签）
- 标签搜索框 — 1000+ 个标签也能直接搜
- 键盘导航（↑↓ 选 / 回车打开 / 空格预览 / x 选中）
- 多选 + 批量操作（改重要度 / 标噪声 / 删除）

### 📅 时间线 `/v2/`
按时间倒着排列所有记忆，每天一张卡片，点开看那天的详情。右侧有迷你时间线可以快速跳到很久以前。

### ✦ 记忆星图 `/v2/network/`
把所有记忆画成一个星空，相关的记忆之间有连线。
> 这个视图没怎么打磨，可能有 bug 和疏漏。

### ⚙ 控制台 `/v2/console/`
管理类操作的入口，四个子页，各有用处：

- **导入工作台** — 把对话 / 笔记 / 文件导入成记忆。可视化看到拆分过程、实时进度、累计花费；支持试跑（只解析前几段先看效果）；失败能看到 LLM 原文。
- **配置** — 切换 / 管理多组 LLM profile（即时生效不重启）；调衰减权重参数；直接编辑 6 个 prompt，可单条 / 全部恢复默认，或一键对齐原作者版本。
- **Breath 模拟** — 输入一句话，看记忆系统会检索 / 浮现出哪些记忆、怎么打分排序；也能看检索打分微调、"哪条常被想起"的统计、最近搜索追溯。
- **回收站** — 软删除的记忆在这里恢复，或彻底清除。

### 📱 手机端 `/v2/mobile/`
独立的手机网页，可"添加到主屏幕"当 App 用。定位是**简化便捷版**——常用的随手能做，重活留给桌面：
- 4 个 tab：记忆首页 / 审阅 / 日历 / 设置
- 浏览、筛选、搜索、编辑、单条手动写入、重新脱水、情感唤起、回收站、调衰减与检索打分参数
- **桌面有、手机暂没有**：记忆星图、记忆格网格视图、合并、完整导入工作台（手机仅支持基础文本 / 文件导入）

## 2. 工作流工具

### 🪄 导入工作台 — 可视化拆分

导入时能看到全过程：

- **试跑模式** — 只解析前 N 段，先看看效果，不用怕一上来花太多 API 钱
- **实时进度** — 当前处理到第几段、累计花了多少美元
- **失败原因可见** — LLM 输出有问题时能看到原文，知道为什么挂了
- **支持 Claude Exporter 浏览器插件 JSON 格式** — 直接导入你和 Claude 的对话
- **多种来源** — 拖拽文件、粘贴文本、上传 ZIP 都行
- **跨天对话不再塌陷**（按 turn 的时间各自归位，而非塌成一个时刻）
- **chunk 内每行嵌入时间戳前缀**（`[2026-05-02 19:07] [用户] ...`）— 让 DIGEST_PROMPT 能给每条 item 准确推断 event_time，发生时间精确对应到具体段落，而不是整 chunk 共用一个时间戳

### 🔄 重新脱水 — 不满意可以重做

当 LLM 写的标题 / 摘要 / 标签 / 正文不满意时：

- **一键让 LLM 重做**（默认只动元数据，正文不变）
- **勾选"同时重写正文"** — 用原始对话彻底重提炼正文
- **预览 → 编辑 → 接受 三步走**：
  1. 左右双栏对比旧版本和新版本
  2. 觉得不行可以编辑或重做
  3. 满意才真正写入
- 改完不满意时，原文还在，可以再来

技术细节：用了"主题锚点法"，防止 LLM 把整段原文概括成流水账。

### 🤝 合并预览 — 防误合

系统检测到两条相似记忆建议合并时：
1. LLM 先写一份合并后的草稿
2. 给你看预览，可以编辑、重做、或拒绝
3. 接受才真正合并

### 📋 批量操作

桌面记忆格里：
- 多选记忆（点复选框或按 x 选中）
- 一次性改一批的重要度
- 一次性标噪声 / 取消噪声
- 一次性删除

### 🗑 软删除 + 回收站

- 删除 = 移到回收站
- 设置 → 回收站 看到所有"删过"的
- 可以恢复，也可以"彻底删除"清空

### 🔌 多个 LLM 一键切换

- 配置页加多组 API profile（一组 Claude / 一组 Gemini / 一组 DeepSeek...）
- 一键切换激活的 profile，即时生效不用重启
- "测试连接"按钮看 key 还有效吗、网络多快

### ✏️ Prompt 直接改 — 不用动代码

在网页里直接编辑 LLM 的系统提示词：
- 6 个 prompt 全部可视化编辑
- 单条恢复默认 / 全部恢复默认
- "⇆ 对齐原作者版本" 按钮 — 一键切回上游 prompt

## 3. 智能筛选

### 主题域 (Domain)
数据里一直存着 domain（比如"恋爱""内心""工作"）。所有视图（桌面 timeline / 记忆格 / 手机）都有 domain 筛选，可多选 AND。

### 标签 (Tag)
- **桌面**：能搜 1000+ 个标签
- **手机**：点"🏷 标签"打开底部抽屉，搜索 + 多选
- 已选的永远显示在最上面（不会找不到怎么取消）

### 状态筛选
- 全部 / ★ 钉决 / ✦ 重要 / ❀ Feel / 已内化 / 待消化 / 我写的 / ⌀ 噪声

### 全文搜索
- 标题 / 摘要 / 正文 / 标签 都搜
- 桌面按 `/` 键快速聚焦

### 情感唤起罗盘 (Mood Evoke) — ⚗️ 实验性
- 在一个 2D 心情坐标里选（消极 ↔ 积极 × 平静 ↔ 激动）
- 找出最匹配那种心情的记忆
- LLM 用这种心情口吻把它们串成一段叙事
- ⚗️ **这是个还不太成熟的功能，没认真测过，可能有 bug**

# 第二部分：涉及到原作者代码的改动（透明披露）

下面是涉及到底层代码的改动，尽量解释为什么需要。如果你担心和原作者版本不一致，看完就知道每条都做了什么。

## A. Bug 修复（建议回流上游）

### 1. 时区统一 UTC
原版后端混用本地时间和 UTC，在中国 / 日本等时区会差 8-9 小时。新版统一存 UTC，前端按你的时区显示。

### 2. 置顶桶可以标噪声
原版置顶 (`protected`) 的桶重要度被锁在 10，想标"噪声"（改成 1）会被静默忽略。新版自动放开锁，让你能标。

### 3. 中文引号防 JSON 解析失败（鲁棒性建议）
LLM 生成 JSON 时如果对话里有引号，容易因为引号嵌套导致 JSON 解析挂。新版在 prompt 里加了一条"用中文引号「」"做兜底。

## B. 字段重命名（无法回退）

这部分是早期重构，新版前端到处用这些新字段，无法回退。但**完全兼容原版数据**（老桶自动迁移）。

- `pinned` 拆成 `protected`（防衰减）+ `highlight`（浮现优先）— 原本一个字段管两件事容易混
- `digested` 改名 `internalized` — 跟"已归档"语义混
- 新增 `event_time`（事件实际发生时间，跟数据库写入时刻分开）
- 新增 `created_by`（"ai" / "user"，区分谁写的）

## C. Prompt（双轨制）

| Prompt | 上游有 | 代码默认 |
|---|---|---|
| `DEHYDRATE` | ✓ | 上游原版 |
| `DIGEST` | ✓ | 在上游基础上加了功能性规则（查看原文锚点 / 事件时间推断 / 重要度量表 / 防 LLM 保守打分）。注：重要度量表是按"重视亲密关系语境"调的，会让打闹 / 承诺 / 袒露类记忆分数偏高——见第二部分末尾说明 |
| `MERGE` | ✓ | 精炼过的版本（强调"沉淀成一条"而非拼接、反流水账、越合越短）。上游原版可一键对齐 |
| `REDEHYDRATE` | ✗ | 本 fork 新增（上游没有这个功能） |
| `REGEN_CONTENT` | ✗ | 本 fork 新增（主题锚点法，上游没有） |
| `ANALYZE` | ✓ | 上游原版 |

配置页"⇆ 对齐原作者版本"按钮一键把 DEHYDRATE / DIGEST / MERGE / ANALYZE 切回上游版。本 fork 新增的 REDEHYDRATE / REGEN_CONTENT 保留（上游没有，无可对齐）。

> 这套 prompt 在 DeepSeek 上实测过，不一定适合所有人——可在配置页里直接改，或一键对齐原作者版。

## D. 新增的 API endpoint

下面这些都是"加法"，不改原来的逻辑，只是新增能力：

- 重新脱水：`/api/bucket/{id}/redehydrate` + `/redehydrate-commit`
- 软删除：`/api/bucket/{id}/delete` + `/restore` + `/purge`
- 回收站：`/api/trash`
- API profile 管理：`/api/config/api/...`
- 衰减配置：`/api/decay-config` + `/reset`
- Prompt 编辑：`/api/prompts-config` + `/align-upstream` + `/reset`
- 自动备份：`/api/backup`
- Embedding 补缺：`/api/embeddings/backfill`

## E. 检索与排序精度（中文场景优化）

原版检索在"中文长正文 + 关键词"场景下有些召回 / 排序短板，本 fork 做了一组优化。**这些只影响"搜索 / 浮现时哪些记忆被选中、怎么排序"，不改衰减 / 做梦 / feel / 记忆桶 / 情感权重这些核心机制。**

**默认生效（开源版你拿到就是这样）：**

- **关键词命中优先** — 任一字段命中你的搜索词，该记忆就会被选出，不会因为时间久、重要度低而被埋掉（上游是综合分过阈值才出）。
- **检索新近权重对齐上游** — 搜索排序里"偏向最近"的程度保持上游的温和值，老记忆不会在搜索里被快速埋掉。
- **中文分词更准** — 内置 jieba 分词，长句搜索不再失准。
- **feel（私密感受）只走专门入口** — 不参与普通关键词 / 浮现检索，只能通过 `domain='feel'` 读取。
- **噪声（你标过的）默认不出现在检索里**（可手动包含）。
- **移除了上游的"自动解决"** — 低重要、30 天没动的记忆不再被系统自动标记完成，这类记忆存活期因此更长。

**可选的高级检索旋钮（默认全关 = 跟上游一致，配置页可开）：** 严格关键词模式（jieba token 命中）、标题命中加分、关键词优先排序、温度偏置（让情感更暖的记忆更易浮现）、**正文权重 `content_weight`**（默认 1.0 对齐上游；调高到 3.0 让"正文里写过的内容"也能被搜到）。开了才改变召回，不开无影响。

**关于重要度评分（importance）：** 它按"重视亲密关系"的语境调过——打闹、玩梗、承诺、袒露这类会给得比中性记忆高一些。所以某些记忆分数偏高是有意的，不是 bug。配置页可一键切回原作者的评分方式。

**关于批量导入：** 导入是 LLM 提炼，可能漏掉个别片段（尤其用较弱的模型时），同一份内容导入两次也会有差异——属正常，重要内容可手动补录。导入质量受 prompt 影响，可在配置页自行调优。

# 第三部分：每天自动备份（防丢失）

新版自带一套备份机制：
- GitHub Actions 每天北京时间 03:00 自动运行
- 把所有 buckets/ 推到你的私人 GitHub 仓库
- 用纯 Python git（不需要系统装 git）
- 针对环境变量错配等导致持久盘数据丢失的风险（此类事故确有先例，曾丢失约一周记忆）

部署时设 `OMBRE_BACKUP_REPO` / `OMBRE_BACKUP_TOKEN` 环境变量即可，详见 DEPLOY.md「自动备份配置」。

## 如何反向同步原作者更新

```bash
git remote add upstream https://github.com/P0luz/Ombre-Brain.git
git fetch upstream
git log dcba1eb..upstream/main --oneline   # 看上游有什么新 commit
git cherry-pick <commit>                    # 把想要的拿过来
```

（本仓库已配好 upstream remote，fork 后可直接 git fetch upstream）
